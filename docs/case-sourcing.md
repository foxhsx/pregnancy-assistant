# 案例来源与匿名化工作流 (Case Sourcing & Anonymization)

## 概述

本文档定义了孕期教育网站 MVP 的案例来源流程、匿名化处理规范和伦理审查要求。所有真实案例必须经过本流程处理以保护隐私并确保合规。

**适用范围**：所有需要真实案例支持的内容  
**目标**：在保护隐私的前提下，提供有价值的真实案例参考

---

## 1. 案例来源分类

### 1.1 可接受的案例来源

| 来源类型 | 要求 | 优先级 |
|----------|------|--------|
| 公开去识别化数据 | 来自公开数据集、已脱敏 | 高 |
| 已发表案例研究 | 学术期刊发表的案例，带知情同意 | 高 |
| 公开新闻报道 | 媒体公开报道（需核实准确性） | 中 |
| 志愿者分享 | 签署知情同意书的自愿分享 | 中 |
| 合作机构提供 | 医院/机构提供（需伦理审批） | 低（仅后期） |

### 1.2 禁止使用的来源

- 未经授权的医院内部数据
- 社交媒体未公开分享的内容
- 匿名论坛/帖子的原始内容（无可追溯来源）

## 2. 匿名化处理流程

### 2.1 匿名化步骤

```
Step 1: 识别个人信息 (PII)
    ↓
Step 2: 脱敏处理
    ↓
Step 3: 验证脱敏效果
    ↓
Step 4: 记录脱敏过程
    ↓
Step 5: 审批发布
```

### 2.2 需要移除/替换的信息

| 信息类型 | 处理方式 | 示例 |
|----------|----------|------|
| 姓名 | 完全移除或替换为假名 | "张三" → "孕妈小李" |
| 日期 | 泛化或移除精确日期 | "2024年3月15日" → "孕中期" |
| 地点 | 移除或泛化 | "北京市朝阳区" → "一线城市" |
| 医院/机构 | 移除具体名称 | "XX医院" → "某三甲医院" |
| 特殊事件 | 泛化描述 | 具体病症详情 → 症状类别 |
| 联系方式 | 完全移除 | 电话、邮箱等 |

### 2.3 脱敏技术

- **泛化**：将精确值转换为范围（年龄 28 → 20-30）
- **抑制**：完全移除敏感字段
- **假名化**：使用假名替换真实身份
- **数据扰动**：添加随机噪声（数值型数据）

### 2.4 脱敏效果验证

在发布前，必须验证：
- [ ] 无法通过案例识别具体个人
- [ ] 无遗漏的 PII 信息
- [ ] 案例仍有参考价值（不过度脱敏）

## 3. 知情同意流程

### 3.1 知情同意要求

所有真实案例必须获得知情同意，同意内容应包括：

**同意书内容**：
1. 案例使用目的说明
2. 将如何呈现案例（是否会修改、如何脱敏）
3. 发布范围（网站、出版物等）
4. 参与者权利（随时撤回同意）
5. 联系方式

### 3.2 同意流程

```
1. 说明目的
   ↓
2. 提供书面说明
   ↓
3. 回答疑问
   ↓
4. 签署同意书
   ↓
5. 保存记录
   ↓
6. 后续跟踪（可随时撤回）
```

### 3.3 同意书模板

```
【案例分享知情同意书】

一、分享目的
您分享的孕期经历将被用于孕期教育网站，帮助其他孕妇了解孕期知识。

二、使用方式
您的经历将被匿名化处理后，以案例形式展示在网站上。

三、隐私保护
- 您的真实姓名不会被公开
- 具体的日期、地点将被泛化处理
- 任何可识别您身份的信息都会被移除

四、您的权利
- 您可以随时撤回同意
- 撤回后，相关内容将立即从网站移除

五、联系方式
如有疑问，请联系：[联系方式]

签署人：_________ 日期：_________
```

## 4. 伦理审查

### 4.1 需要伦理审批的情况

- 使用国内医院提供的案例数据
- 使用涉及特定疾病/状况的敏感案例
- 与外部机构合作获取的案例

### 4.2 伦理审批流程

```
1. 准备审批材料
   - 案例数据使用申请
   - 知情同意书模板
   - 隐私保护方案
   
2. 提交伦理委员会
   
3. 伦理委员会审核
   
4. 获得审批/修改后重新提交
   
5. 记录审批结果
```

### 4.3 伦理审查要点

- 数据来源是否合法
- 知情同意是否充分
- 隐私保护措施是否有效
- 案例使用是否会造成伤害

## 5. 案例质量标准

### 5.1 内容要求

- [ ] 案例内容与孕期教育主题相关
- [ ] 包含可借鉴的经验或教训
- [ ] 科学性准确（有证据支持）
- [ ] 语言流畅、易懂

### 5.2 格式要求

- [ ] 包含案例概述
- [ ] 关键要点清晰
- [ ] 匿名化处理完成
- [ ] 引用来源标注

### 5.3 合规要求

- [ ] 知情同意书签署
- [ ] 伦理审批（如需要）
- [ ] 脱敏记录完整

## 6. 案例库管理

### 6.1 案例分类

| 类别 | 说明 |
|------|------|
| 饮食安全 | 孕期饮食相关案例 |
| 健康管理 | 孕期健康问题处理 |
| 心理支持 | 情绪与心理案例 |
| 家庭关系 | 家庭支持相关案例 |
| 分娩经历 | 生产过程分享 |

### 6.2 案例存储

- 原始数据：加密存储，限定访问权限
- 匿名化后数据：可公开访问
- 同意书：安全存储，保留至少 5 年

### 6.3 更新与维护

- 定期审查现有案例的时效性
- 更新过时信息
- 移除无效/不相关案例

## 7. 常见问题

### Q1: 公开新闻报道的案例可以直接使用吗？

A: 需要核实准确性，并进行匿名化处理。最好能找到原始来源进行交叉验证。

### Q2: 匿名化后还能保证案例的参考价值吗？

A: 关键是保留核心信息（时间线、关键决策、结果），而非具体细节。泛化处理后的案例仍具有参考价值。

### Q3: 志愿者可以直接分享经历吗？

A: 需要签署知情同意书，并说明使用方式和隐私保护措施。
